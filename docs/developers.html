Build note

Build Configuration

PocketInkOS uses arduino-cli to install the ESP32 board core, install project libraries, and compile the sketch. Install Arduino CLI first using the official Arduino installation guide.

Firmware uploads are handled by esptool v5.x. Install esptool using the official Espressif installation guide. Before building, find the connected serial port and update PORT in bin/config.sh.

bin/app.sh generates compile wrappers for nested source files, builds src/src.ino, verifies that the app binary exists, and uploads it with esptool.

Network apps are enabled by default. Build with ENABLE_NETWORK_APPS=0 when you want a local-only image without Wi-Fi, feeds, weather, or AI.

Install Arduino core and libraries
bin/all_libs.sh

Installs the ESP32 Arduino core plus the project libraries used by the firmware.

Find the connected port
ls /dev/cu.usbmodem* /dev/cu.usbserial* 2>/dev/null

Use the matching device path as PORT in bin/config.sh.

Edit device target
$EDITOR bin/config.sh

Set PORT before compiling or uploading. Keep the ESP32-S3 FQBN unless the board target changes.

Build and upload
bin/app.sh

Compiles the firmware for the configured ESP32-S3 board and uploads the app binary with esptool.

Build and upload without network apps
ENABLE_NETWORK_APPS=0 bin/app.sh

Compiles without optional network app sources and without the Network menu category.

Arduino CLI

Install arduino-cli before running project scripts. Arduino documents Homebrew, install-script, and binary-download paths for Linux and macOS.

Arduino CLI installation
esptool

Install esptool v5.x before uploading firmware. Espressif documents pip, virtual environment, and binary-release installation paths.

esptool installation
Device target

bin/config.sh sets PORT, UPLOAD_SPEED, FQBN, and ESPTOOL. On macOS the port usually appears under /dev/cu.usbmodem* or /dev/cu.usbserial*; on Linux it usually appears under /dev/ttyACM* or /dev/ttyUSB*.

Credential setup

Copy .env.example to .env, then edit .env with local values when network apps or contact metadata need runtime credentials.

Wi-Fi

Set WIFI_SSID, WIFI_PASSWORD, and WIFI_DISPLAY_NAME for the Wi-Fi app and time sync.

Contact card

Feel free to set all the contact details if you want to use the device as a business card.

Weather

Set WEATHER_LATITUDE and WEATHER_LONGITUDE for Weather app to link your location.

AI

Set AI_API_BASE_URL, AI_API_KEY, AI_MODEL_PRIMARY, and AI_MODEL_SECONDARY for the AI app.

Hardware layer

Drivers Included

The firmware wraps hardware access behind small system modules. Application code draws through AppDisplay, reads touch through TouchInput, and receives button events through SmartButton.

E-paper display

200x200 monochrome e-paper driven over SPI2 through epaper_driver_display; supports full refresh, partial refresh, and a 5 KB framebuffer.

Touch controller

FT6336-compatible touch over I2C at address 0x38; a 5 ms FreeRTOS poll task queues down, move, and up events.

Buttons

BOOT_BUTTON_PIN on GPIO0 and PWR_BUTTON_PIN on GPIO18 are debounced through SmartButton with single, double, and long-press events.

Environment

SHTC3 ambient temperature/humidity on the shared I2C bus plus ESP chip temperature through the available ESP-IDF temperature sensor API.

Clock

DeviceClock keeps local time in firmware. PCF85063 RTC support exists behind ENABLE_RTC_CLOCK and is disabled by default.

Audio

ES8311 codec setup and I2S PCM16 capture at 16 kHz are used by the AI voice path.
